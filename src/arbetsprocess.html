Information om arbetsprocess

Vid användning av Github skapades två branches utöver main, dessa var testing och development. Testing var avsedd för att fungera vid nya implementeringar som endast skulle undersökas innan de valdes in i koden eller inte. Development ska innehålla utvecklingen av hemsidan, det vill säga utkastet innan publicering där majoriteten av arbetet görs. När korrigeringar gjorts i koden som ska sparas så görs detta via commits till Github. Filerna som inte skulle versionshanteringas, det vill säga node_modules, package-lock.json, dist/ samt .parchel-cache, las i mappen .gitignore enligt instruktionerna bifogades till kursen. Det tog dock lite tid att lära sig hur processen fungerade vilket i sin tur resulterade i en hel del "onödiga" commits ibland.

Kommandot npm init användes i terminalen för att skapa ett nytt NPM-projekt. Parcel installerades till NodeJs-projektet genom att skriva npm install parcel --save-dev och lades därefter till i node_modules-filen.

För att minifiera CSS och HTML samt transpilera och minifiera Javascript användes Parcel. JavaScript skulle även omvandlas till en version som de flesta webbläsare stödjer.

Jag upplevde svårigheter att otpimera JPG-filer tills jag till slut testade att istället skriva jpeg på filerna, och då fungerade det.

Browserlist fungerar och anpassar sig för att stödja flera olika webbläsare. Genom att ange kommandot npx browserslist så skrev konsolen ut en lista över webbläsare som matchade det kriterie jag definierat i min package.json, dvs "> 0.5%, last 2 versions, not dead". Parcel använde sina egna inbyggda inställningar för att transpilerera JavaScript-koden baserat på webbläsarmålen som definierats i browserslist
